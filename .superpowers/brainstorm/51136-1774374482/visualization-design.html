Layer 3: Interactive Graph Visualization

A parameterized Databricks notebook that renders attack path graphs via vis.js

Notebook: notebooks/gold/05_investigation_graph.py

Parameters via dbutils.widgets:

trigger_ip — malicious IP (e.g., "185.220.101.1")
hostname — target host (optional — auto-discovered from auth match)
time_window_hours — how far after trigger to look (default: 24)
catalog_name — Unity Catalog name (default: security_poc)

Notebook Flow

Cell 1: Parameters → Cell 2: %run 00_timeline_common → Cell 3: Find trigger auth event
→ Cell 4: Build timeline → Cell 5: Derive graph nodes + edges → Cell 6: Render vis.js

Edge Types

━━ ssh_login ━━→ malicious IP → host (red, thick)
━━ executed ━━→ user → process/command (blue)
━━ escalated ━━→ user → user (pink, dashed, animated)
━━ modified ━━→ user → account (yellow)
━━ system ━━→ user → system event (gray, thin)

Interactive Controls (rendered as HTML buttons above the graph)

Toggle buttons filter event categories. Clicking a node shows detail panel with timestamp, raw event, and relevance score. Hover shows tooltip summary.

Implementation: All rendering happens in a single displayHTML() cell. The notebook collects timeline data via PySpark, converts to JSON, injects into an HTML template containing vis.js CDN + the graph config. No additional infrastructure required.
